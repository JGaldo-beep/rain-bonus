# This is a depth-first demo for technical hiring, not a production system

The rain-driven courier bonus system is built as a **portfolio/demo piece whose audience is hiring engineers** — people who will both read the code and watch it run. We therefore optimize for a single airtight vertical slice (rain forecast → supply/demand gap → recommended COP bonus per zone, with explainability) running on **synthetic data**, and for a defensible model and clean architecture — *not* for production concerns. Explicitly out of scope unless they make the demo more convincing: live Rappi internal APIs, real auth/multi-tenant API keys, ML forecasting (Prophet/LSTM), NOAA ingestion, weekly retraining, and prod monitoring (these are the SPEC's Phase 3–4). Depth beats breadth: better one slice an engineer can trust than three half-wired phases.
